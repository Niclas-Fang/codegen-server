# 代码补全服务器部署指南

## 项目概述

基于Django的代码补全API服务器，为VSCode智能编码助手插件提供后端服务。支持 RAG（检索增强生成）通过代码知识库提升补全质量。

## 系统要求

- Python 3.14.1+
- Pixi 包管理器
- 至少一个 API 密钥: `DEEPSEEK_API_KEY`、`ZHIPU_API_KEY`、`OPENAI_API_KEY` 或 `ANTHROPIC_API_KEY`

## 快速开始

### 1. 安装依赖

```bash
pixi install
export DEEPSEEK_API_KEY="your-api-key"
```

### 2. 初始化数据库

```bash
pixi run python manage.py migrate
```

### 3. 启动服务

```bash
pixi run python manage.py runserver
```

### 4. 验证

```bash
pixi run python test_api.py
```

## RAG 配置

### 什么是 Graph-RAG

Graph-RAG（基于图的检索增强生成）是传统 RAG 的增强版本。它不仅使用语义相似度检索代码片段，还构建了代码的**知识图谱**，通过图遍历发现相关代码：

- **传统 RAG**：只找语义相似的代码
- **Graph-RAG**：找语义相似 + 调用关系 + 继承关系 + 导入依赖的代码

Graph-RAG 特别适合代码补全场景，因为代码高度结构化，函数调用链、类继承关系携带了大量上下文信息。

### 环境变量

| 变量 | 默认值 | 说明 |
|-----|-------|------|
| `RAG_ENABLED` | `true` | 全局启用/禁用 RAG |
| `GRAPH_RAG_ENABLED` | `true` | 全局启用/禁用 Graph-RAG |
| `RAG_EMBEDDING_MODEL` | `sentence-transformers/all-MiniLM-L6-v2` | 嵌入模型 |
| `RAG_EMBEDDING_CACHE_SIZE` | `1000` | Embedding 缓存大小 |
| `GRAPH_HOPS` | `2` | 图遍历跳数 |

### RAG 特性

- **Graph-RAG**：语义搜索 + 图遍历混合检索，发现调用链和继承关系
- **增量索引**：只更新变更的文件，无需全量重建
- **相似度阈值**：默认 0.5，低于此值的结果会被过滤
- **Embedding 缓存**：LRU 缓存，避免重复计算
- **项目隔离**：不同项目使用独立的向量存储和知识图谱
- **自动提取 includes**：如果未提供 includes，自动从 prompt 中提取

## 生产部署

### 环境变量

```bash
DEEPSEEK_API_KEY=your-api-key  # 或 ZHIPU_API_KEY / OPENAI_API_KEY / ANTHROPIC_API_KEY
DEBUG=False
SECRET_KEY=<随机字符串>  # 必须使用随机密钥，禁止使用占位符
ALLOWED_HOSTS=your-domain.com
RAG_ENABLED=true  # 如需 RAG 功能
```

### 使用 Gunicorn

```bash
pixi run gunicorn api.wsgi:application --bind 0.0.0.0:8000 --workers 4 --timeout 120
```

## 常见问题

### 服务无法启动

```bash
pixi run python manage.py check
```

### API 返回错误

```bash
# 检查 API 密钥是否设置
echo $DEEPSEEK_API_KEY

# 测试直接连接
curl http://127.0.0.1:8000/api/v1/completion
```

### RAG 检索不生效

```bash
# 检查索引是否存在
pixi run python -m completion.rag.indexer stats

# 如未构建索引，先构建
pixi run python -m completion.rag.indexer index <代码目录>

# 检查项目路径是否正确
# project_path 必须与索引时使用的 --project-path 一致
```

### 索引占用空间过大

- 默认使用 `all-MiniLM-L6-v2` 模型（384维）
- 每个 chunk 约占用 1.5KB 存储空间
- 大项目可通过调整 `CHUNK_SIZE` 减少 chunk 数量
